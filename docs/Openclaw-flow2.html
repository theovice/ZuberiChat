KILO
OpenClaw Runtime — orchestration engine, policy gate, MCP gateway
Runs on KILO. The control plane. Holds your secrets, permissions, audit logs, and tool registry.
Loads relevant Skills (playbooks) for this task type into the LLM context
Registers available Plugins (local) and MCP servers (remote on M710q-B or VPS)
Runs the Policy Gate — checks permissions, data privacy rules, write-action confirmation requirements
Decides model routing: does this task touch private data? → Ollama CPU. Otherwise → Cloud API
Manages the full loop: LLM → tool call → result → LLM → repeat until done
Logs every tool call at the MCP Gateway level: agent, tool, inputs, outputs, timestamp
▼
task + skills + tool list → Model Router
KILO
Model Router — decides which LLM handles this
Hard rules enforced by the Runtime before any token is sent to a model.
Routing Decision Table
☁ Cloud API (OpenRouter / Claude / GPT-4o)
Planning, reasoning, complex decisions
Code generation (DevAgent tasks)
Research + summarization (public data)
Any task where context has NO private data
🔒 Ollama CPU (KILO — no GPU, never leaves machine)
Any context containing local file paths
Email or calendar content
Personal identifiers or private project data
Anything you explicitly mark as local-only
Hard rule: If the context contains private data — RTX 3060 is NOT used. Ollama runs on CPU cores only, keeping the GPU 100% free for gaming. Cloud APIs never see private data. No exceptions.

All nodes
Tool Execution Layer — run by Runtime on KILO, dispatched to remote nodes as needed
Three tool types. All return results back to the LLM the same way. Location varies by risk tier.
Skills
Playbooks loaded into LLM context before it starts. Not called — they guide how the LLM approaches the task.
research_workflow repo_push_sop schedule_pattern competitor_analysis_template
Plugins(KILO — local, fast)
In-process tool calls. Low-risk, trusted, no network hop. Run directly on KILO.
git.clone git.commit git.push fs.write formatter.report
MCP Servers(M710q-B or VPS)
Remote, isolated tool servers. Risky or heavy operations run here, never on KILO directly.
mcp://browser/search mcp://github/read mcp://rag/query mcp://scheduler/queue
▼
tool results back to LLM → loop continues until task complete
M710q-A
Always-On Persistence Layer — memory, scheduling, state
M710q-A runs 24/7. Headless. Controlled from KILO over Tailscale. Max RAM to 32GB.
MemoryAgent + Qdrant — long-term vector DB; indexes past sessions, decisions, Examiner.Sable context
SchedulerAgent + Redis — task queue for deferred + recurring jobs (e.g. weekly competitor refresh)
Survives KILO being powered off — scheduled tasks run whether you're at your desk or not
Gateway logs archive — every tool call logged here for audit and debugging
▼
scheduled task queued on M710q-A · browser research dispatched to M710q-B
M710q-B
Always-On Execution Layer — browser, ops, isolated tools
M710q-B runs 24/7. Handles anything you don't want running on KILO directly. Isolated by design.
BrowserAgent (Playwright) — web research, scraping, competitor analysis for Examiner.Sable
OpsAgent — Docker container health, disk monitoring, alerts for all three nodes
Compromise here does not touch KILO — secrets, keys, and the Runtime stay isolated on your primary machine

Micro-Loop — This Task Step by Step
"Research competitors, push report to repo, schedule weekly refresh"
CLI
You → CLI on KILO
Type the request. CLI passes it to the Runtime with session context.
RT
Runtime → Model Router (KILO)
No private data in this task. Routes to cloud LLM.
→ Cloud API (Claude / GPT-4o via OpenRouter)
LLM
Cloud LLM → Runtime (tool request)
Reads task + research_workflow Skill. Decides first step: research competitors.
mcp://browser/search({ query: "Examiner.Sable competitors wholesale real estate 2025" })
B
Runtime → BrowserAgent MCP (M710q-B)
Dispatched over Tailscale to M710q-B. Playwright runs the search, returns results. KILO untouched.
→ Returns: [ {title: "PropStream", url: "..."}, {title: "DealMachine", ...}, ... ]
LLM
Cloud LLM → Runtime (next tool request)
Has research results. Asks MemoryAgent if we've analyzed these before.
mcp://rag/query({ q: "previous competitor analysis Examiner.Sable" })
M
Runtime → MemoryAgent (M710q-A)
Qdrant query over Tailscale. Returns any prior analysis context.
→ Returns: "Previous analysis from 2025-01-14: PropStream reviewed, DealMachine not yet covered"
LLM
Cloud LLM → Runtime (write report)
Has all data. Drafts report. Requests file write + git push.
fs.write({ path: "docs/competitor-analysis.md", content: "..." })
git.commit({ msg: "add competitor analysis report" }) → git.push()
RT
Runtime → Local Plugins (KILO)
fs.write and git.push run locally on KILO. Fast, trusted, no network hop.
→ Returns: { written: true, pushed: true, commit: "a3f92c1" }
LLM
Cloud LLM → Runtime (schedule weekly refresh)
Task says "schedule weekly refresh." Queues it on SchedulerAgent.
mcp://scheduler/queue({ task: "competitor_research", cron: "0 9 * * MON" })
S
Runtime → SchedulerAgent (M710q-A)
Queued in Redis on M710q-A. Will fire every Monday at 9AM — even if KILO is off.
→ Returns: { scheduled: true, next_run: "2026-02-23T09:00:00" }
✓
Final Output → CLI on KILO
LLM produces confirmation. Runtime hands to CLI. Printed to your terminal.
→ "Done — competitor analysis pushed to docs/competitor-analysis.md (commit a3f92c1). Weekly refresh scheduled every Monday 9AM."
