task + session context → Runtime
KILO
OpenClaw Runtime — orchestration, policy, MCP gateway
Runs on KILO. Control plane — holds secrets, permissions, audit logs, tool registry.
Loads relevant Skills (playbooks) into LLM context before execution starts
Registers Plugins (local) and MCP servers (remote on M710q-B)
Runs Policy Gate — checks permissions, data privacy classification, write-action confirmations
Routes to Model Router: private data? → Ollama GPU on KILO. Complex reasoning? → Cloud API
Manages the full loop: LLM → tool call → result → LLM → repeat until done
Every tool call logged at MCP Gateway: agent, tool, inputs, outputs, timestamp
▼
task + skills + tool list → Model Router
KILO
Model Router — decides which LLM handles this request
Two backends. Choice is made by data sensitivity and task complexity — not availability.
Routing Decision Table
☁ Cloud API (Claude / GPT-4o via OpenRouter)
Complex planning and multi-step reasoning
High-quality code generation (DevAgent tasks)
Research + summarization using public data
Tasks requiring frontier model quality
🔒 Ollama GPU (RTX 3060 — fast, stays on KILO)
Any context containing local file paths
Email or calendar content
Personal identifiers or private project data
Fast tool-call classification and quick tasks
Anything marked local-only by policy
RTX 3060 12GB advantage: Runs 7B models at Q4_K_M entirely in VRAM at ~40–50 tokens/sec. Private data never leaves KILO. Cloud APIs handle complexity. Both backends active simultaneously — no conflict.

All nodes
Tool Execution Layer — run by Runtime, dispatched to correct node by risk tier
Three tool types. All return results back to the LLM the same way. Location varies by risk.
Skills
Playbooks loaded into LLM context before execution. Not called — they guide the LLM's approach.
research_workflow repo_push_sop schedule_pattern competitor_analysis
Plugins(KILO)
In-process calls. Low-risk, fast, trusted. Run directly on KILO — no network hop.
git.clone git.commit git.push fs.write formatter.report
MCP Servers(M710q-B)
Remote, isolated tool servers over Tailscale. Risky or heavy ops run here, not on KILO.
mcp://browser/search mcp://github/read mcp://rag/query mcp://scheduler/queue
▼
tool results → LLM → loop until task complete
M710q-A
Always-On Persistence Layer — memory, scheduling, state
M710q-A runs 24/7. Headless, controlled from KILO over Tailscale. Upgrade to 32GB RAM.
MemoryAgent + Qdrant — long-term vector DB; indexes past sessions, decisions, Examiner.Sable project context
SchedulerAgent + Redis — task queue for deferred and recurring jobs (weekly competitor refresh, cron tasks)
Survives KILO being off — scheduled tasks fire whether you're at your desk or not
Audit log archive — every tool call stored here for debugging and replay
▼
browser research dispatched to M710q-B · schedule queued on M710q-A
M710q-B
Always-On Execution Layer — browser, ops, isolated tools
M710q-B runs 24/7. Handles anything you want isolated from KILO. Controlled over Tailscale.
BrowserAgent (Playwright) — web research, scraping, competitor analysis for Examiner.Sable
OpsAgent — Docker health, disk monitoring, resource alerts across all three nodes
Compromise here stays isolated — secrets, Runtime, and GPU inference remain on KILO

Micro-Loop — This Task Step by Step
"Research competitors, push report to repo, schedule weekly refresh"
CLI
You → CLI on KILO
Type the request. CLI passes it to the Runtime with session context.
RT
Runtime → Model Router (KILO)
No private data in this task. Research + planning → routes to Cloud API.
→ Cloud API (Claude / GPT-4o via OpenRouter)
LLM
Cloud LLM → Runtime (tool request)
Reads task + research_workflow Skill. First step: search competitors.
mcp://browser/search({ query: "Examiner.Sable competitors wholesale real estate 2025" })
B
Runtime → BrowserAgent MCP (M710q-B)
Dispatched over Tailscale to M710q-B. Playwright runs the search. KILO untouched.
→ Returns: [ {title: "PropStream", ...}, {title: "DealMachine", ...}, ... ]
LLM
Cloud LLM → Runtime (memory query)
Checks if we've analyzed these competitors before.
mcp://rag/query({ q: "previous competitor analysis Examiner.Sable" })
M
Runtime → MemoryAgent (M710q-A)
Qdrant query over Tailscale. Returns prior context.
→ Returns: "Previous analysis 2025-01-14: PropStream reviewed. DealMachine not yet covered."
LLM
Model Router switches → Ollama GPU (KILO)
Report writing will reference local file paths. Router hard-switches to local GPU model.
→ Ollama on RTX 3060 · ~40–50 tok/s · data never leaves KILO
fs.write({ path: "docs/competitor-analysis.md", content: "..." })
RT
Runtime → Local Plugins (KILO)
fs.write runs locally. Then git commit + push.
git.commit({ msg: "add competitor analysis" }) → git.push()
→ Returns: { written: true, pushed: true, commit: "a3f92c1" }
LLM
Cloud LLM → Runtime (schedule job)
Back to cloud for scheduling decision. No private data involved.
mcp://scheduler/queue({ task: "competitor_research", cron: "0 9 * * MON" })
S
Runtime → SchedulerAgent (M710q-A)
Queued in Redis on M710q-A. Fires every Monday 9AM — even if KILO is off.
→ Returns: { scheduled: true, next_run: "2026-02-23T09:00:00" }
✓
Final Output → CLI on KILO
Task complete. Confirmation printed to your terminal.
→ "Done — competitor analysis pushed (commit a3f92c1). Weekly refresh scheduled every Monday 9AM."
